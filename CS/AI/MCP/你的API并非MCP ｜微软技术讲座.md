自动生成MCP的缺点
- 选择太多，上下文太厂
- 接口描述文档未针对性优化
- LLM 的MCP重在实现目标，而传统的API重在让程序员组合功能实现目标

如何正确的编写MCP
- 先自动化生成
	- 在删减
- 补全功能描述
- 增加有意思的工具
- 写凭验证文件
